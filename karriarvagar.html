Sex vanliga vägar — researchade och faktakollade

Karriärvägar inom IT

"IT" är inte ett jobb — det är hundra olika jobb. Här är sex vanliga roller: vad de faktiskt gör, vad som krävs på nybörjar-, medel- och avancerad nivå, vilka verktyg som verkligen används 2026, och konkreta labbar per spår. Allt nedan är researchat mot färska källor — inte gissat.

Alla roller ser olika ut beroende på företag, bransch och ort. Det här är en överblick som pekar dig i rätt riktning, inte hela sanningen. Läs riktiga jobbannonser i ditt närområde också — det är där du ser exakt vad som efterfrågas just nu, inte i en lista på nätet.

Hela kartan — vad leder till vad

Allt börjar på samma ställe: AZ-900 eller Google IT Support. Det är grunden för i princip varenda gren nedan, oavsett vart du vill sluta. Vissa vägar kräver att du tar första jobbet på vägen — andra går att hoppa rakt in i. Båda sätten finns med.

AZ-900 eller Google IT Support — grunden för nästan allt
Vanligaste vägen: första jobbet som IT-support / Helpdesk

Alternativet — hoppa direkt: forskningen är tydlig på en punkt: SOC-analytiker är en genuint egen ingång. Företag anställer folk med noll professionell erfarenhet och lär upp dem på plats — Security+ eller SC-900 räcker som grund, inget krav på helpdesk-år först. De flesta andra roller på den här kartan (pentester, arkitekt, senior-roller) kräver däremot 1–7 års erfarenhet någonstans i kedjan innan de blir realistiska — de nås bäst via första jobbet, inte runt det.

Sammanfattat, ärligt: de allra flesta av dessa vägar kräver ingen högskoleexamen — SOC-analytiker, DevOps, molntekniker, utvecklare och projektledare rekryteras idag på certifikat, portfolio och erfarenhet snarare än papper. Tre roller är undantagen värda att känna till: Nätverksarkitekt, QA Architect och Data Architect ser oftare en examen som "standard expectation" — men i samtliga tre fall är det ändå destinationer efter många års erfarenhet, inte startpunkter, så det är inget som stoppar dig från att börja idag och avgöra saken senare.

Mönstret upprepas i varje gren: bred grund, specialiserat certifikat, ett labb som faktiskt bevisar det, och till sist rollen. Skillnaden är bara hur många år som krävs innan nästa steg blir realistiskt — SOC-analytiker är öppen direkt, andra kräver att du samlar erfarenhet först. Alla vägar bär dit, bara i olika takt.

🎧 IT-support / Helpdesk

Första kontakten när något krånglar — användarsupport, felsökning, kontohantering. Den bredaste och mest tillgängliga ingången till hela branschen, och grunden hela den här sajten redan bygger på.

Fullt beskrivet på Vägkartan och Certifikat-sidan — Google IT Support eller AZ-900 är starten. Nästan alla andra spår nedan går att nå därifrån.

🌐 Nätverkstekniker

Bygger, driftar och felsöker nätverken allt annat vilar på — kontor, datacenter, molnanslutningar.

Nybörjare

Grunderna

  • TCP/IP, subnetting, OSI-modellen
  • Grundläggande routing och switching
  • CCNA — branschens erkända startpunkt
Medel

Verklig drift

  • CCNP (Enterprise Core + inriktning)
  • Felsöka hybridmiljöer: on-prem + AWS VPC + Azure VNet
  • Python och Ansible för automatisering
Avancerat

Arkitektur

  • CCIE eller motsvarande
  • SD-WAN, multi-vendor (Cisco, Juniper, Arista)
  • Terraform för nätverk som kod

Verktyg som faktiskt används: Wireshark, SolarWinds/PRTG, Splunk, Python, Ansible, Terraform.

Certifikatstege: CCNA → CCNP → CCIE.

Labbtips: bygg ett litet nätverk gratis i Cisco Packet Tracer eller GNS3 — konfigurera VLAN och routing, felsök trafiken live med Wireshark. Dokumentera som ett labb i ditt CV, precis som CV-guiden visar.

☁️ Molntekniker / Cloud Engineer

Bygger och driftar infrastruktur i molnet istället för i ett eget serverrum — ett av de snabbast växande och bäst betalda spåren just nu.

Nybörjare

Grunderna

  • Nätverksgrunder: IP, DNS, brandväggar
  • Linux-kommandoraden
  • AZ-900 eller AWS Cloud Practitioner
Medel

Bygga på riktigt

  • AZ-104 eller AWS Solutions Architect Associate
  • Virtuella nätverk, lagring, IAM-behörigheter
  • Infrastructure as Code med Terraform
Avancerat

Specialisering

  • Säkerhet, DevOps eller arkitektur-inriktning
  • Bygga in säkerhet från grunden (IAM, kryptering)
  • Multi-cloud-medvetenhet

Verkligheten på marknaden: AWS har störst marknadsandel (~31 %), Azure närmar sig snabbt (~28 %, och används av över 95 % av Fortune 500-företagen), Google Cloud ligger på ~12 %. Nästan 9 av 10 företag använder fler än ett moln. Välj en plattform och gå djupt innan du breddar dig — ytlig kunskap i tre moln väger mindre än verklig kompetens i ett.

Dominerande verktyg: Terraform — molnoberoende och används oavsett vilken plattform du väljer.

Certifikatstege: AZ-900 (redan din startpunkt på den här sajten!) → AZ-104 eller AWS SAA → Terraform Associate.

Labbtips: skapa ett gratis-tier-konto (sätt alltid en fakturaalarm först), sätt upp en virtuell maskin och en lagringsresurs för hand — bygg sedan om exakt samma sak med Terraform-kod. Den övningen är själva kärnan i rollen.

⚙️ DevOps-ingenjör

Bygger broarna mellan utveckling och drift — automatiserar allt som annars görs manuellt. Ett av de högst betalda spåren, och ett av de mest efterfrågade 2026.

Nybörjare

Grunderna

  • Linux och kommandoraden
  • Git och GitHub
  • Bash-scripting, sedan grundläggande Python
Medel

Automation

  • CI/CD (GitHub Actions är gratis och vanligast att börja med)
  • Docker — bygga images, skriva Dockerfiles
Avancerat

Skala upp

  • Kubernetes — pods, deployments, services
  • Terraform för infrastruktur som kod
  • Övervakning: Prometheus/Grafana

Fakta som spelar roll: Docker och Kubernetes är i praktiken ett krav för medel- och seniorroller 2026 — inte "nice to have". Terraform dominerar infrastructure-as-code oavsett molnplattform.

Mest efterfrågade certifikat: AWS Solutions Architect Associate, Certified Kubernetes Administrator (CKA), HashiCorp Terraform Associate.

Realistisk tid: med konsekvent studerande når de flesta anställningsbar nivå på 6–9 månader.

Labbtips: containerisera en egen liten app med Docker, bygg en enkel CI/CD-pipeline med GitHub Actions som testar och deployar den automatiskt, kör sedan samma app i en gratis lokal Kubernetes-sandbox (t.ex. minikube).

🔒 Cybersäkerhet / SOC-analytiker

Övervakar, upptäcker och stoppar attacker. Global brist på kompetens — miljontals lediga tjänster världen över.

Nybörjare

Grunderna

  • Nätverks- och OS-grunder
  • Security+ — universellt erkänt, nämns i över 70 % av jobbannonserna
  • eller SC-900 — Microsofts säkerhetsgrund, starkt i Microsoft-tunga miljöer (vanligt i Sverige)
  • TryHackMe: Pre-Security-spåret (kräver ingen erfarenhet)
Medel

Analys

  • CySA+ för analytiskt djup
  • Fler TryHackMe-spår, egna CTF:er
Avancerat

Bevisa det

  • HackTheBox-utmaningar
  • Egen GitHub-portfolio med skript och "writeups"

Den viktigaste fakta i hela researchen: certifikat ensamt räcker inte här — arbetsgivare vill uttryckligen se TryHackMe- eller HackTheBox-erfarenhet och en dokumenterad portfolio. Det är exakt samma princip som resten av den här sajten bygger på: labb och bevis slår ett papper ensamt.

Certifikatstege: Security+ → CySA+, eller Microsoft-spåret SC-900 → SC-200 (Security Operations Analyst) — särskilt relevant i svenska bolag som redan kör Microsoft 365/Azure, där SC-200 direkt matchar SOC-analytiker-rollen.

Löneläge (internationellt, som fingervisning): SOC-analytiker på entry-nivå ligger normalt lägre än erfarna specialister — precis som i alla IT-spår klättrar lönen snabbt med bevisad erfarenhet.

Labbtips: börja gratis på TryHackMe:s Pre-Security-spår, lös en utmaning och skriv en kort "writeup" (vad du gjorde, vad du lärde dig) — publicera den på GitHub. Det är precis den portfolion arbetsgivare efterfrågar.

📊 Data Engineer / Data Analyst

Data Analyst besvarar affärsfrågor och bygger dashboards. Data Engineer bygger rörledningarna som förflyttar och strukturerar datan i botten. Olika jobb, samma startpunkt.

Data Analyst

6–9 månader

  • SQL och Excel — dagliga verktyg
  • Power BI eller Tableau
  • Grundläggande Python och statistik
Data Engineer

8–12 månader

  • SQL på djupet (window functions, CTE:er)
  • Python för pipelines
  • Ett molnplattform + orkestreringsverktyg
Bevisa dig

Projekt slår cert

  • Inget certifikat dominerar det här spåret
  • 2–4 riktiga end-to-end-projekt väger tyngre

Verktyg som faktiskt används: SQL och Python i nästan varje jobbannons. Analytiker lägger till Power BI/Tableau och Excel; ingenjörer lägger till molnplattform och orkestreringsverktyg.

Labbtips: öva SQL gratis (t.ex. SQLZoo eller Mode), bygg en Power BI-dashboard av egen data (din träningslogg, dina studietimmar — vad som helst), och som nästa steg: ett litet end-to-end-pipelineprojekt som hämtar, rensar och lagrar data automatiskt.

🖥️ Systemadministratör

Håller servrar, Active Directory, Windows och Linux-miljöer igång — själva ryggraden i ett företags IT.

Redan väl täckt av grunderna på den här sajten: AZ-900 och Windows/Linux-kunskaperna i Vägkartan är exakt startpunkten. Nästa steg brukar vara fördjupning i Active Directory, PowerShell-automation och virtualisering (Hyper-V/VMware) — se labbidéerna för konkreta övningar.
